# Backend Notes

Architecture decisions and Spring Boot 4 / Java 25 quirks relevant to this codebase.

---

## Java & Spring versions

- **Java 25** with `toolchain { languageVersion = JavaLanguageVersion.of(25) }` in `build.gradle.kts`.
- **Spring Boot 4** requires Java 17+ minimum; we target 25 for virtual threads and newer APIs.
- Spring Security 7 ships with Boot 4 — the `HttpSecurity` DSL uses lambda-style configuration only (no deprecated chained methods).

## Entity conventions

All entities follow this pattern:

```java
@Entity
@Table(name = "some_table")
@Getter @Setter @NoArgsConstructor
public class SomeEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    private UUID id;

    // ... fields ...

    @Column(nullable = false, updatable = false)
    private Instant createdAt;

    @PrePersist
    void prePersist() { this.createdAt = Instant.now(); }
}
```

Key rules:
- Primary key: `UUID`, generated by Hibernate (`GenerationType.UUID` delegates to RFC 4122 random UUIDs).
- `createdAt` set in `@PrePersist` — never passed in from the client.
- No Lombok `@Data` (avoids accidental `equals`/`hashCode` on mutable JPA entities).

## DTOs as records

All request/response objects are Java `record`s:

```java
public record FeedbackRequest(
    @NotNull FeedbackCategory category,
    @NotBlank @Size(min = 10, max = 2000) String message
) {}
```

Records are immutable by design — no setters needed, Jackson deserialises them via the canonical constructor.

## Database conventions

| Convention | Value |
|---|---|
| Primary key type | `UUID PRIMARY KEY DEFAULT gen_random_uuid()` |
| Timestamps | `TIMESTAMPTZ` (always UTC) |
| User table name | `app_user` (avoids conflict with reserved word `user` in PostgreSQL) |
| Enum storage | `VARCHAR` with a `CHECK` constraint listing valid values |
| Soft deletes | Not used — hard deletes with `ON DELETE CASCADE` where appropriate |

## Flyway migrations

Migrations live in `src/main/resources/db/migration/` and are named `V{n}__{description}.sql`.

Current sequence:

| Version | Description |
|---|---|
| V1 | Create `app_user` table |
| V2 | Create `refresh_token` table |
| V3 | Create `groups` and `group_members` tables |
| V4 | Create `group_invites` table |
| V5 | Create `availability` table |
| V6 | Create `meetings` table |
| V7 | Create `feedback` table |

Never edit a committed migration — create a new one instead.

## Auth flow

1. `POST /api/v1/auth/signup` → creates user, returns `AuthResponse` (access + refresh tokens).
2. `POST /api/v1/auth/signin` → validates credentials, returns `AuthResponse`.
3. `POST /api/v1/auth/refresh` → validates refresh token, issues new access token.
4. All protected endpoints expect `Authorization: Bearer <accessToken>`.
5. `@AuthenticationPrincipal UserPrincipal` injects the authenticated user into controller methods.

## Security config

Spring Security 7 lambda DSL example:

```java
http
    .csrf(AbstractHttpConfigurer::disable)
    .sessionManagement(s -> s.sessionCreationPolicy(STATELESS))
    .authorizeHttpRequests(auth -> auth
        .requestMatchers("/api/v1/auth/**").permitAll()
        .anyRequest().authenticated()
    )
    .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);
```

## Testcontainers

Integration tests use `@SpringBootTest` + Testcontainers to spin up a real PostgreSQL 16 container. Tests are ordered with `@TestMethodOrder(MethodOrderer.OrderAnnotation.class)` and share one container instance per class via `@Container` + `@DynamicPropertySource`.

Requires Docker daemon at runtime. In CI (GitHub Actions) Docker is available by default. Local developers need Docker Desktop or equivalent.

## Error handling

A `@RestControllerAdvice` class maps exceptions to RFC 7807-style JSON:

```json
{ "code": "VALIDATION_ERROR", "message": "...", "details": { "field": "reason" } }
```

Common codes: `VALIDATION_ERROR`, `NOT_FOUND`, `CONFLICT`, `UNAUTHORIZED`, `FORBIDDEN`.
